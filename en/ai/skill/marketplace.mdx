The skills market is the discovery entry point for reusable skills. It combines featured skills from template configuration with searchable repository indexes from connected skill repositories.

## Open the Skills Market

Open **Explore / Skills** to browse available skills.

The page has two common views:

- **Featured skills**: curated skills configured from `skills-market.yaml`.
- **My Skills**: skills installed in your default workspace, with quick access to the workspace skills workbench.

If no skill repositories are available, ask an administrator to configure repositories in **Settings / Tenant / Skills**.

## Browse Featured Skills

Featured skills highlight recommended packages for common work. A featured card can include a badge, title, description, avatar, publisher, tags, source repository, version, license, and market signals such as stars or downloads.

Open a skill card to review:

- what the skill is for
- publisher and source repository
- skill ID and version
- tags and license
- source link, when available

## Search Repository Indexes

Repository indexes are generated by syncing connected skill repositories. Search by name, skill ID, publisher, or tags to find skills that are not shown in the featured section.

Supported repository sources include:

- `github` repositories that contain skill folders with `SKILL.md`
- `clawhub` registries
- the system-managed `workspace-public` organization repository

## Install or Update a Skill

To install a skill:

1. Open a skill detail page or card.
2. Select **Install or Update**.
3. Choose the target workspace when prompted.
4. Confirm the installation.

If the skill is already installed in that workspace, Xpert reuses the existing installation. If a newer indexed version is available, the installation is updated in place.

After installation, open **Workspace / Skills** to inspect the files, edit supported text files, or enable the skill for an agent.

## Share a Workspace Skill

Skills that were uploaded directly into a workspace can be shared to the organization market. Repository-installed skills are not shared from a workspace because their source of truth remains the connected repository.

When you share a workspace skill, Xpert publishes a snapshot to the organization public skill repository. Later local file edits do not automatically update the shared version; use **Update Share** when you want to republish.
