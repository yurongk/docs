title: 管理技能

管理员负责维护租户或组织的技能供给，包括仓库注册、公共技能发布、模板同步和默认工作区技能。

## 租户技能页面

进入 **Settings / Tenant / Skills** 管理组织级技能。

管理员可以在这里：

- 初始化系统托管的公共技能仓库
- 上传 ZIP 技能包到公共仓库
- 注册技能仓库
- 查看同步后的仓库索引
- 同步模板技能资产
- 在需要时强制执行完整模板同步

## 公共技能仓库

Xpert 使用名为 `workspace-public` 的系统仓库提供方承载组织内部共享技能。

它主要用于：

- 将工作区上传的技能分享到组织市场
- 将模板技能包发布为组织可见技能
- 让其他工作区安装组织内部创建的技能

通常由管理员在租户技能页面初始化该仓库。

## 仓库来源

技能仓库定义了索引技能的来源。

| Provider | 用途 |
| --- | --- |
| `github` | 从 GitHub 仓库同步技能，可配置 URL、分支和可选目录路径。 |
| `clawhub` | 从 ClawHub 注册中心同步公开或官方技能。 |
| `workspace-public` | 系统托管的组织技能市场，用于共享工作区技能。 |
| ZIP/local upload | 直接上传本地技能目录压缩包。 |

仓库注册后需要同步，技能才会出现在市场搜索中。

## 模板技能同步

模板技能来自外部模板资产目录。关键文件包括：

| 文件或目录 | 用途 |
| --- | --- |
| `skills-market.yaml` | 定义技能市场中的精选技能和筛选标签。 |
| `skill-repositories.yaml` | 声明租户级默认技能仓库。 |
| `workspace-defaults.yaml` | 声明新用户默认工作区应安装的默认技能。 |
| `skill-packages/*/bundle.yaml` | 将本地内置技能发布到公共仓库。 |
| `skill-packages/*/SKILL.md` | 本地技能的实际指令和资源文件。 |

日常使用 **Sync Template Skills** 执行增量同步。需要重新检查仓库、索引、内置公共技能、精选引用和工作区默认技能时，使用 **Force Full Sync**。

## 默认工作区技能

默认工作区技能会在新用户个人默认工作区创建时自动安装。对应技能引用需要能够在 `skills-market.yaml` 和仓库索引中解析。

适合配置为默认技能的能力包括文档、表格、代码、演示文稿等大多数用户都可能使用的基础能力。

## 运营建议

- 仓库凭据应放在仓库配置中，不要写入技能文件。
- 共享到组织市场的工作区技能是快照；源技能更新后需要重新分享。
- 环境变量或模板文件变化不会自动回填已有租户；已有租户需要管理员手动同步或注册对应仓库。
