---
title: 管理工作区技能
sidebar_position: 3
---

工作区技能工作台用于管理某个工作区内已安装的技能。它既适合日常启用技能，也适合对轻量技能进行编辑和维护。

## 打开工作区技能

进入某个工作区后，打开 **Skills**。

工作台会展示：

- 当前工作区已安装的技能
- 技能名称、摘要、来源、发布者、可见性、标签和仓库信息
- 当前选中技能的文件树
- 支持文本文件的预览和编辑区

## 安装技能到工作区

你可以通过多种方式添加技能。

| 方式 | 适用场景 |
| --- | --- |
| Add Skill from GitHub | 已知 GitHub 仓库地址、分支和可选目录路径。 |
| Upload Skills | 手中已有包含一个或多个技能目录的 ZIP 文件。 |
| Register from Repository | 需要连接或浏览仓库来源，再从索引中安装技能。 |
| Bulk Import | 需要从某个仓库批量导入多个索引技能。 |

上传 ZIP 时，可以包含一个技能，也可以包含多个技能目录。每个技能目录都必须包含 `SKILL.md`。

## 查看和编辑技能文件

选择一个技能后，可以浏览它的文件树。工作台支持加载、预览、下载、上传、保存和删除技能包中的文件。

常见操作包括：

- 调整 `SKILL.md` 指令
- 添加参考资料或示例
- 更新脚本或辅助文件
- 增加模板或素材资源

内置编辑器主要支持文本类文件。二进制素材仍然可以在文件树中上传、下载和管理。

## 删除技能

当工作区不再需要某个技能时，可以删除对应安装。删除会移除工作区中的技能副本，并刷新已安装技能列表。

如果该技能已经被分享到组织市场，删除来源技能时也会按照组织共享流程移除或停用对应市场索引。

## 推荐流程

1. 从市场、GitHub 或 ZIP 上传安装技能。
2. 在 **Workspace / Skills** 打开技能。
3. 检查 `SKILL.md` 和辅助文件。
4. 如果技能归工作区所有，按需要编辑指令或资源。
5. 在智能体中启用技能，或将其分享到组织市场。
