本文用于从开关反查它控制的功能和入口。feature 开关只决定模块或入口是否启用，permission / roleName 仍需要同时满足。

## 功能开关对应功能列表

这个表用于从 feature 开关反查它控制的具体功能。feature 开关只决定模块或入口是否启用；表中列出的权限或角色门槛仍然需要同时满足。

| Feature 开关 | 控制的功能 / 入口 | 主要路径或入口 | 额外 permission / roleName 门槛 |
| --- | --- | --- | --- |
| `FEATURE_SETTING` | Settings 根页面和设置模块容器 | `/settings` | 内部菜单仍按各自权限控制。 |
| `FEATURE_HOME` | Home feature 父级和首页功能容器 | `home` | 当前主菜单 dashboard 入口已注释；Home 下的 Analytics 子功能另由 `FEATURE_HOME_CATALOG`、`FEATURE_HOME_TREND` 控制。 |
| `FEATURE_DASHBOARD` | Dashboard 子功能定义 | Home / Dashboard | 默认开启，但当前主菜单 dashboard 子入口已注释。 |
| `FEATURE_ORGANIZATION` | Manage Organization feature 定义 | `organizations` | 组织列表、详情、创建、编辑、删除实际仍由 `ALL_ORG_VIEW`、`ALL_ORG_EDIT`、`ORG_USERS_VIEW`、`ORG_USERS_EDIT` 控制。 |
| `FEATURE_USER` | Users 页面、用户列表、用户搜索、用户详情、用户创建/删除/更新 | `/settings/users` | 根据操作不同，需要 `ALL_ORG_VIEW`、`ALL_ORG_EDIT`、`ORG_USERS_VIEW` 或 `ORG_USERS_EDIT`。 |
| `FEATURE_USER` | Groups 页面和用户组维护 | `/settings/groups` | 查看需要 `ORG_USERS_VIEW`；维护需要 `ORG_USERS_EDIT`。 |
| `FEATURE_USER` | Invite 按钮、Invite 列表、重发邀请、删除邀请 | Users 邀请区域 | 查看需要 `ORG_INVITE_VIEW` 或 `ORG_INVITE_EDIT`；维护需要 `ORG_INVITE_EDIT`。 |
| `FEATURE_ROLES_PERMISSION` | Roles 页面和角色权限矩阵 | `/settings/roles` | `CHANGE_ROLES_PERMISSIONS`。 |
| `FEATURE_INTEGRATION` | System Integration 页面和集成维护 | `/settings/integration` | `INTEGRATION_EDIT`。 |
| `FEATURE_EMAIL` | Email feature 父级和邮件管理入口 | `settings/email-history` | 邮件模板由子开关 `FEATURE_EMAIL_TEMPLATE` 和权限 `VIEW_ALL_EMAIL_TEMPLATES` 控制；邮件历史对应 `VIEW_ALL_EMAILS`。 |
| `FEATURE_EMAIL_TEMPLATE` | Email Template 页面 | `/settings/email-templates` | `VIEW_ALL_EMAIL_TEMPLATES`。 |
| `FEATURE_FILE_STORAGE` | File Storage 子功能定义 | `settings/file-storage` | `FILE_STORAGE_VIEW`；当前 Settings 菜单里未列出单独入口时，需要结合具体页面确认入口是否开放。 |
| `FEATURE_SMTP` | Custom SMTP 页面 | `/settings/custom-smtp` | `CUSTOM_SMTP_VIEW`。 |
| `FEATURE_SMS_GATEWAY` | SMS Gateway 子功能定义 | `tasks/me` | 枚举和环境开关存在，但当前默认 feature 定义中该子项被注释，没有默认 feature 记录。 |
| `FEATURE_COPILOT` | AI Copilot 设置、Copilot 创建/更新/启停、Copilot 统计、Provider、用户管理 | `/settings/copilot` | `COPILOT_EDIT`；`COPILOT_VIEW` 只覆盖单独使用它的查看/使用场景。 |
| `FEATURE_COPILOT_KNOWLEDGEBASE` | Copilot 知识库列表和统计 | Copilot 所属知识库入口 | `KNOWLEDGEBASE_EDIT`。 |
| `FEATURE_COPILOT_CHAT` | 默认 feature 定义中的 Copilot Chat 子功能 | `chat` | 还取决于实际聊天入口和页面使用的权限。 |
| `FEATURE_XPERT` | Chat 主入口和 Common Assistant | `/chat`、`/chat/x/common` | `CHAT_VIEW`。 |
| `FEATURE_XPERT` | Explore Xperts 和 Xpert Workspace 入口 | `/explore`、`/xpert` | `XPERT_EDIT`；进入工作区后继续受 workspace capabilities 控制。 |
| `FEATURE_XPERT` | Common Assistant、Workspace Assistant 的 Assistants 设置 | `/settings/assistants` | 页面入口需要 `SUPER_ADMIN` 或 `ADMIN`；租户默认配置编辑需要 `SUPER_ADMIN`。 |
| `FEATURE_XPERT` | Xpert 列表、下拉选项、统计、全量 workspace 列表、workspace 读/运行/写/管理、view extensions | 后端和 workspace API | 根据操作不同，需要 `XPERT_EDIT`、`CHAT_VIEW` 或计算出的 `workspace.capabilities.*`。 |
| `FEATURE_XPERT_CLAWXPERT` | ClawXpert 聊天入口和 ClawXpert assistant 绑定 | `/chat/clawxpert`、用户级 ClawXpert 绑定 | 需要父级 `FEATURE_XPERT`；聊天入口在本矩阵中没有额外枚举权限。 |
| `FEATURE_XPERT_CHATBI` | ChatBI 聊天入口、Chat 侧边栏 ChatBI、ChatBI assistant 配置、Chat BI 设置 | `/chat/chatbi`、`/settings/chatbi` | 需要父级 `FEATURE_XPERT`；设置页还需要 `MODELS_EDIT`，并可能同时需要 `FEATURE_MODEL`。 |
| `FEATURE_XPERT_CODEXPERT` | Chat 侧边栏 CodeXpert 外链入口 | Chat 侧边栏 | 需要父级聊天访问；本矩阵中没有额外枚举权限。 |
| `FEATURE_XPERT_DEEP_RESEARCH` | Chat 侧边栏 DeepResearch 外链入口 | Chat 侧边栏 | 需要父级聊天访问；本矩阵中没有额外枚举权限。 |
| `FEATURE_MODEL` | Data / Semantic Model 入口和语义模型操作 | `/data/models` | 入口和编辑需要 `MODELS_EDIT`；XMLA 查询需要 `MODELS_VIEW`；后端列表 API 需要 `ADMIN`。 |
| `FEATURE_MODEL` | Data Sources 页面和数据源操作 | `/settings/data-sources` | `DATA_SOURCE_VIEW` 或 `DATA_SOURCE_EDIT`。 |
| `FEATURE_MODEL` | Chat BI 设置 | `/settings/chatbi` | 与 `FEATURE_XPERT` 同时使用；权限门槛是 `MODELS_EDIT`。 |
| `FEATURE_PROJECT` | Data / Project 入口和 BI project/story 编辑能力 | `/data/project` | `STORIES_EDIT`。 |
| `FEATURE_STORY` | Story feature 定义 | `/story` | 当前矩阵中 `/story` 页面访问使用 `STORIES_VIEW`；未列出显式主导航 feature 开关。 |
| `FEATURE_BUSINESS_AREA` | Business Area 设置 | `/settings/business-area` | `BUSINESS_AREA_EDIT`。 |
| `FEATURE_INDICATOR` | Indicator App 基础 feature、指标查看/编辑能力 | `/indicator-app`、指标相关区域 | `INDICATOR_VIEW`、`INDICATOR_EDIT` 或 `INDICATOR_MARTKET_VIEW`。 |
| `FEATURE_INDICATOR_APP` | Indicator App 入口 | `/indicator-app` | 需要父级 `FEATURE_INDICATOR` 和 `INDICATOR_MARTKET_VIEW`。 |
| `FEATURE_INDICATOR_MARKET` | Analytics 默认 feature 定义中的 Indicator Market 子功能 | `indicator/market` | 具体操作继续使用指标权限。 |
| `FEATURE_INDICATOR_REGISTER` | Analytics 默认 feature 定义中的 Indicator Creation 子功能 | `indicator/create` | 具体操作继续使用指标权限。 |
| `FEATURE_HOME_CATALOG` | Analytics 默认 feature 定义中的 Home Catalog 子功能 | `catalog` | 本矩阵中未单独列出具体页面权限。 |
| `FEATURE_HOME_TREND` | Analytics 默认 feature 定义中的 Home Trend 子功能 | `trend` | 本矩阵中未单独列出具体页面权限。 |
| `FEATURE_DATA_FACTORY` | Data Factory feature 枚举和 data factory 权限 | Data Factory 区域 | 当前默认 feature 定义没有初始化该开关；依赖前需要结合具体入口确认。 |
