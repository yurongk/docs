任务是 CodeXpert 的工作单元，记录指令、执行上下文、终端输出与变更集，便于复盘与协作。

## 创建任务

启动任务时需要提供：

- 清晰的指令（目标 + 约束）
- 仓库与分支上下文
- 可选的附件或参考资料

系统会在所选环境中执行任务。

## 任务生命周期

任务可能处于以下状态：

- `queued`: 排队等待
- `running`: 正在执行
- `idle`: 等待下一步
- `failed`: 执行失败
- `archived`: 已归档

## 任务记录内容

每个任务包含：

- 指令、状态与时间戳
- 仓库与分支信息
- 沙盒终端输出
- 一个或多个变更集

## PR 操作

基于最新变更集可以：

- 创建 PR
- 打开已有 PR
- 复制 patch 或 git apply

## 归档任务

完成审阅后可归档任务，保持当前列表聚焦。归档任务仍可随时打开查看。

## 最佳实践

- 指令写清目标与约束，减少歧义。
- 关键步骤后同步，形成清晰变更集。
- 开 PR 前先审阅 diff。
