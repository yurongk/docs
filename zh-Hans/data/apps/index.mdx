插件应用是运行在 UOSE 系统中的业务型插件。它不是单个工具或单篇提示词，而是一组可以被安装、授权、配置和审核的业务能力集合，通常包含工作台视图、助手工具、业务助手模板、配置项、权限声明和可追溯的数据闭环。

UOSE 的插件应用市场用于展示优秀的官方应用、合作伙伴应用和社区应用。用户可以按业务板块发现应用，管理员可以根据组织权限启用应用，开发者可以用 Xpert 插件体系把行业流程封装成可复用的业务应用。

## 插件应用通常包含什么

| 能力 | 说明 |
| --- | --- |
| 工作台视图 | 在 UOSE 工作台中提供审核、维护、配置、上传、复核或运营页面。 |
| 助手工具 | 通过 Agent 中间件暴露可被助手调用的业务动作，例如保存解析结果、执行匹配、查询资料库。 |
| 业务助手模板 | 提供开箱即用的助手配置、提示词、工具权限和起始问题。 |
| 权限与操作声明 | 声明 read、write、admin 等操作边界，便于平台做授权和审计。 |
| 配置表单 | 暴露管理员可维护的资源 ID、召回策略、阈值、默认选项或外部系统配置。 |
| 业务数据模型 | 保存可复核、可重试、可追踪的结构化数据和证据。 |
| 知识库或本体集成 | 复用 UOSE 知识库、资源图谱、本体快照和 Agent 工具。 |

## 常见业务板块

插件应用会按业务板块分组展示。常见板块包括：

| 板块 | 适合的应用类型 |
| --- | --- |
| 生产制造 | BOM、工艺、质量、设备、生产计划、技术协议和图纸处理。 |
| 供应链 | 采购、供应商协同、库存、物流、到货验收和交付跟踪。 |
| 财务管理 | 费用审核、应收应付、预算、报销、合同付款和财务分析。 |
| 销售服务 | 商机、报价、合同、客户跟进、售后服务和项目交付。 |
| 客户服务 | 工单、知识库问答、服务质量、客户反馈和问题升级。 |
| 人力资源 | 招聘、入职、培训、绩效、政策问答和员工服务。 |
| 风险合规 | 审批、审计、合规检查、风控规则和异常处置。 |
| 数据分析 | 指标治理、数据质量、业务看板、语义查询和分析助手。 |

## 如何选择插件应用

选择插件应用时，建议先确认：

- 应用是否面向当前业务板块和真实流程。
- 是否提供工作台视图，方便人工复核和运营。
- 助手工具是否有明确调用顺序、输入结构和证据要求。
- 是否声明权限边界，区分普通使用、写入操作和管理员能力。
- 是否能复用 UOSE 知识库、本体资源、审计和组织权限。
- 是否提供官方或合作伙伴维护说明，便于升级和排障。

## 如何开发插件应用

开发 UOSE 插件应用时，应优先使用 AI 产品中的插件开发体系，而不是把业务逻辑散落在单个提示词或临时脚本里。推荐阅读：

- [插件开发](/zh-Hans/ai/plugin/plugin-development)
- [插件核心概念](/zh-Hans/ai/plugin/core-concepts)
- [插件配置表单](/zh-Hans/ai/plugin/schema-ui-extension)
- [远程组件插件开发](/zh-Hans/ai/plugin/remote-component)
- [插件发布和使用](/zh-Hans/ai/plugin/publish-and-use)
- [开发自定义智能体应用](/zh-Hans/ai/tutorial/develop-custom-agentic-app)

一个面向 UOSE 的业务应用通常需要完成：

1. 定义插件元信息、图标、目标产品和应用市场展示内容。
2. 声明 `targetApps: ['data-xpert']` 和 `targetAppMeta`，说明它提供的视图、工具、业务应用、模板和权限能力。
3. 注册服务端模块，提供实体、服务、Agent 中间件和工作台视图提供器。
4. 把核心业务动作拆成结构清晰、可重试、可审计的助手工具。
5. 为需要人工复核的流程提供工作台视图或远程组件。
6. 提供业务助手模板，让用户可以一键创建可用的助手。
7. 配置权限、默认值、发布方式和升级说明。

开发时要把“Agent 自动化”和“人工复核”设计成同一个闭环：Agent 负责抽取、推理、检索和执行；工作台负责查看、确认、修正、回写和审计。这样插件应用才能成为可运营的业务系统能力，而不是一次性的自动化脚本。
