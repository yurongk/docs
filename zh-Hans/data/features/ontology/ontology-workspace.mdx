---
title: 本体空间
description: 跨资源查看 ontology snapshot、搜索实体并钻取资源图谱。
---

本体空间是 UOSE系统的语义总览工作区。它把各个资源详情中的本体能力提升为跨资源视图，帮助用户查看哪些资源已经完成语义发布、当前图谱是否健康、实体能否被搜索和定位。

## 工作区目标

本体空间回答三类问题：

- 资源状态：当前组织下有哪些资源，哪些已有 current snapshot，哪些发布失败或缺失。
- 实体搜索：用户能否跨资源搜索实体名称、外部键、别名或实体类型。
- 图谱验证：选中资源后能否查看当前本体图谱、节点关系和属性详情。

它是资源接入之后的验收入口，也是 Agent 使用资源之前的语义检查入口。

## Snapshot 健康状态

本体空间按资源展示 snapshot 健康状态：

- ready：当前资源有可用 snapshot 和投影。
- pending：同步或发布处理中。
- publish_failed：本体发布失败。
- projection_failed：实例投影失败。
- missing_snapshot：资源尚未产出 current snapshot。

当资源缺失 snapshot 时，通常需要先执行同步；当发布或投影失败时，应进入资源详情查看同步作业和异常事件。

## 跨资源搜索

本体空间支持按以下维度搜索：

- 实体名称或展示标签。
- 外部键。
- 别名。
- 实体类型。

搜索结果会返回实体所在 resource、实体类型、匹配方式、ontology ID 和基础属性。用户可以把搜索结果作为资源详情图谱定位、Agent target 选择或排障依据。

## 单资源图谱

选择一个已有 snapshot 的资源后，可以查看单资源图谱：

- 节点代表实体实例。
- 边代表关系实例。
- 颜色和图例区分实体类型。
- 详情面板展示实体属性、关系和原始值摘要。

对于大图，系统会启用或允许手动启用超大图模式，减少标签和布局计算，以保持可用性。

## 本体助手

本体空间可以挂载本体助手。对话会携带当前资源、snapshot、图谱版本、选中实体和搜索词，让用户在语义上下文中询问：

- 这个资源里有哪些关键实体类型。
- 为什么某个指标无法执行查询。
- 某个 SAP Entity Set 可以执行哪些动作。
- 当前图谱缺少哪些必要关系。

本体助手依赖 Tenant Ontology Runtime 和 Ontology Assistant 初始化。未初始化时，页面会提示联系管理员。
